开始第一个网页制作吧！

Created2025-02-18|Updated2025-02-18
|Word Count:1.7k|Reading Time:5mins|Post Views:

环境的搭建

我们将要基于Hexo+Github搭建个人博客，本篇博客按照 cutedog的搭建流程以及海拥大佬的美化教程总结而来

安装并配置Node.js

教程 ： nodejs安装和环境配置-Windows（2024）

按照教程配置完成后，在cmd中输入以下命令来检测环境配置的效果

node -v
npm -v

会出现对应工具的版本信息

安装并配置git环境

教程 ： Git 详细安装教程（详解 Git 安装过程的每一个步骤）

安装教程结束后可以在cmd中输入git会出现git工具的指令提示信息或输入

git -v

会出现git工具的版本信息证明安装没有问题
接下来我们需要把本地的git和你的github账户绑定，使用以下的命令

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

生成本地sshkeys密钥，来连接github服务器

可以在vscode中新建终端，或者直接在cmd中输入下面的指令

ssh-keygen -t rsa -C "你的邮箱"

可以在本地c盘user文件夹中看到被隐藏的.ssh文件夹接下来你需要打开.ssh文件夹下的id_rsa.pub，将里面的内容复制，然后进入github账号，在settings下，SSH and GPG keys下new SSH key，title随便取一个名字，然后将id_rsa.pub里的内容复制到Key中，完成后Add SSH Key。

Hexo本地访问博客

我们先按步骤开始。

(1)在D盘根目录创建一个名为Blog的文件夹，文件夹刚被创建时是空的。Hexo框架与以后你自己发布的网页都在这个文件夹中。创建好后，进入文件夹中，按住shift键，右击鼠标点击命令行“Git Bash here”。

(2)使用npm命令安装Hexo，输入:

npm install -g hexo-cli
hexo init myblog #这个操作会在当前文件夹下创造一个名字为myblog的文件夹目录存放初始化的文件和信息
cd myblog
hexo s #启动本地服务器

现在，您应该能够在浏览器中访问 http://localhost:4000 并查看您的 Hexo 博客了。

到目前为止基础的前期准备工作已经完成，接下来将准备开始修改和搭建网站

把本地网站和github服务器连接

首先我们需要创建一个规定好名字的指定仓库，仓库名称为 [你的个人用户名].github.io

接下来要做的就是就是推送网站，也就是发布网站，让我们的网站可以被更多的人访问。在设置之前，需要解释一个概念，在myblog根目录里的_config.yml文件称为站点配置文件，
我们用记事本打开后，拉到底部，将

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
type: ''

这一部分做修改，这一部分时确定连接方式的，需要修改为

deploy:
type: git
repo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .git
branch: main（以前是branch）

相当于添加了服务器地址和分支名称

参考如下：

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
type: git
repository: https://github.com/maets-906/maets-906.github.io.git
branch: main

接下来需要安装git部署插件

npm install hexo-deployer-git --save

接下来我们分别输入三条命令：

hexo clean 
hexo g 
hexo d

之后你的个人博客网站就已经可以被离线访问了

网站的结构调整和美化

结构调整

butterfly美化

官方文档： Butterfly 文檔(一) 快速開始

参考博客： Butterfly 主题美化（一）

主题的安装
npm i hexo-theme-butterfly
改变应用主题

修改 Hexo 根目录下的 _config.yml，把主题改为 butterfly：

安装插件
npm install hexo-renderer-pug hexo-renderer-stylus --save
控制文件迁移

为了减少升级主题后带来的不便，请使用以下方法：

在 hexo 的根目录创建一个文件 _config.butterfly.yml，并把主题目录的 _config.yml 内容复制到 _config.butterfly.yml 去，以后只需要在 _config.butterfly.yml 进行配置就行。

  1. 鉴于每个人的根目录名称都不一样，本帖博客根目录一律以 [Blogroot] 指代。
  2. 本帖涉及魔改源码的内容，会使用 diff 代码块标识，复制时请不要忘记删除前面的 +、- 符号。
  3. 因为 .pug 和 .styl 以及 .yml 等对缩进要求较为严格，请尽量 不要使用记事本等无法提供语法高亮的文本编辑器 进行修改。博主同款编辑器的安装配置教程请参看：Atom 插件安装和推荐
  4. 本帖基于 Butterfly 主题进行魔改方案编写，因此请读者优先掌握 Butterfly 主题官方文档 的内容后再来进行魔改。以避免不必要的兼容性问题。

页面美化

修改代码高亮主题

Butterfly 支持6种代码高亮样式：

  • darker
  • pale night
  • light
  • ocean
  • mac
  • mac light

修改主题配置文件 _config.butterfly.yml

highlight_theme: mac

代码复制功能

修改 主题配置文件
highlight_copy: true

顶部图

配置 解释
index_img 主页的 top_img
default_top_img 默认的 top_img，当页面的 top_img 没有配置时，会显示 default_top_img
archive_img 归档页面的 top_img
tag_img tag 子页面 的 默认 top_img
tag_per_img tag 子页面的 top_img，可配置每个 tag 的 top_img
category_img category 子页面 的 默认 top_img
category_per_img category 子页面的 top_img，可配置每个 category 的 top_img

修改主题配置文件 _config.butterfly.yml

index_img: xxx.png

其它页面 （tags/categories/自建页面）和文章页的 top_img ，请到对应的 md 页面设置 front-matter 中的 top_img

重点总结

大佬文章中的操作我也就不多赘述了，接下来我来讲讲遇到的问题和收获

首先就是图像的分配问题，文章开头的图像和封面的图像在实现过程中遇到了一些麻烦，在这里进行一个总结

首先如果想要修改主页观察到的文章预览的图片我们就要对这个参数进行修改

top_img: /img/touxiang.png
cover: /img/touxiang.png

我们需要具体打开对应的md文件对其中的参数进行修改，top_img负责的是进入文章后标题头的背景图像，cover是外部和标题在一起的图像

接下来是透明度的问题，参考了这个文章设置透明度和字体可以解决问题

接下来是添加新的网页的问题，需要这个指令

hexo new page link

这样就会产生一个文件夹来存放新的页面，然后就可以通过标签页点击进去，使用标签页链接格式
